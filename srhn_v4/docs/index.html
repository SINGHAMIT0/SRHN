Overview & Philosophy

SRHN v4 is not a language model. It is a graph-based associative memory engine that acts as a structured reasoning layer sitting between your application and any local LLM. The graph retrieves relevant knowledge, builds causal chains, detects contradictions, and hands a rich structured context to the LLM — which only needs to handle fluent text generation.

This separation gives you three things no pure LLM approach can: explainability (the chain of activated nodes is fully auditable), learning (Hebbian weight updates, PageRank, episodic memory), and offline operation (the entire system is a 400KB C binary with no Python, no Docker, no GPU required).

Core Concepts

1

Nodes are concepts, not tokens

Each node stores a 384-dim embedding, activation state, PageRank importance, entropy score, and typed connections. Nodes can be Physics laws (with formulas), Code patterns, Emotions, or generic Concepts.

2

Queries propagate activation, not attention

A query is embedded, seeds the HNSW index to find similar nodes, then BFS-propagates activation through the graph with spike decay. The activated subgraph is the "context".

3

The LLM generates, the graph grounds

SRHN builds a structured context string from the activated nodes, chains, and contradictions — then calls the LLM. After generation, it verifies the response against the graph and flags hallucinations.

4

Every query teaches the graph

Feedback signals trigger Hebbian weight updates, contrastive signature learning, and episodic storage. PageRank runs every 100 queries to update node importance. The graph improves over time.

Hello World (C)

c// hello_srhn.c
#include "srhn_v4.h"
#include <stdio.h>

int main(void) {
    /* 1. Create the network */
    SRHNNetwork4 *net = srhn4_create();

    /* 2. Load built-in knowledge bases */
    srhn4_load_physics(net);
    srhn4_load_code(net);

    /* 3. Query */
    SRHNResult4 r = srhn4_query(net, "what is Newton second law");
    printf("Response:   %s\n", r.response);
    printf("Confidence: %.1f%%\n", r.confidence * 100.f);
    printf("Chains:     %u\n", r.n_chains);

    /* 4. Show reasoning chain */
    for (int i = 0; i < r.n_chains; i++) {
        ReasoningChain *c = &r.chains[i];
        printf("  Chain %d: ", i+1);
        for (int j = 0; j < c->length; j++) {
            if (j) printf(" → ");
            printf("%s", c->labels[j]);
        }
        printf("\n");
    }

    /* 5. Feedback */
    srhn4_feedback(net, net->query_counter, 0.9f);

    srhn4_save(net, "/tmp/my_net.bin");
    srhn4_destroy(net);
    return 0;
}

The 19-Step Query Pipeline

Every call to srhn4_query(net, input) runs exactly these steps in order. Understanding this pipeline is the key to extending SRHN.

1

Language Detection

Script-range byte scanning (CJK, Devanagari, Arabic, Cyrillic) + stopword matching for 15 European languages + code keyword detection. Sets result.response_lang.

2

Abbreviation Expansion

Tokenise the query, look up each token in the lexicon's abbrevs[] array, rebuild the expanded string. Enables "ML" → "machine learning".

3

Query Embedding

Call srhn4_embed_text() on the expanded query. Returns a unit-normalised 384-dim signature. Falls back to n-gram hashing if no GloVe/FastText loaded.

4

Cross-Turn Context Blending

Blend current query sig with the last 4 turns using exponential weights (0.25, 0.125, 0.0625, 0.03125). This is what makes "what's the formula?" work after "tell me about Newton's law".

5

Activation Decay

Multiply all node activations by 0.20 before new propagation. Nodes that haven't fired recently return to near-zero. This prevents stale activations polluting the new query.

6

Working Memory Decay

Decay slot salience by 20% per query. Slots below 0.05 are evicted. This simulates short-term forgetting while keeping recently salient nodes primed.

7

HNSW Seed Search

Search the HNSW index with the blended signature. Returns up to 16 seed nodes in O(log n). Falls back to linear scan if HNSW has no results.

8

BFS Propagation

For each seed node, run heap-allocated BFS with spike decay (0.72). Working memory nodes get 10× activation bonus. Inhibitory edges reduce spike strength. Terminates at MAX_HOPS=12 or MAX_ACTIVATED=1024.

9

Hyperedge Evaluation

Scan all hyperedges. Causal edges fire only if all member nodes are active. Associative edges fire on average activation. Fired edges are recorded in result.fired_edges[].

10

Reasoning Chain Building

Find top-8 activated nodes by activation strength, then find all direct and 2-hop connections between them. Build up to MAX_CHAINS=8 labeled chains.

11

Contradiction Detection

Scan all pairs of activated nodes (>0.25 activation) for EDGE_CONTRADICTS connections. Populate result.contradictions[].

12

Top Concept Extraction

Partial-sort activated nodes by activation value, store top 8 labels in result.top_concepts[].

13

Confidence Calibration

Compute raw confidence from resonance + activation + chain count, then run through the isotonic calibration table to produce a real probability.

14

Self-Grow

If best_resonance < selfgrow_thresh (0.18), tokenise the query, filter stopwords, and call srhn4_selfgrow() for each novel token. New nodes connect to nearest HNSW neighbours.

15

LLM Answer (serialised)

Build structured context → select prompt template by dominant node type → call srhn4_llm_generate() → run grounding check → extract new concepts from response.

16

Context Update

Store query + result in ConvContext.turns[]. Update rolling session signature. Insert top activated nodes into working memory.

17

Episodic Storage

Store episode with key nodes and confidence in the 10K circular episodic buffer. Reward field filled in by srhn4_feedback() if called later.

18

Periodic Maintenance

Every 25 queries: srhn4_prune_pass(). Every 100 queries: srhn4_run_pagerank().

19

Metrics

Record latency, compute_used (activated/total), hops_taken. Update rolling average latency. Increment LLM call counter.

API Reference

Core Network API

SRHNNetwork4 * srhn4_create(void)

Allocate and initialise a new SRHN network. Initialises node/edge arrays, HNSW index, episodic memory, RNG, and all pthread locks. Always check for NULL return.

Returns: pointer to heap-allocated network, or NULL on OOM.
void srhn4_destroy(SRHNNetwork4 *net)

Free all memory, close WAL, release LLM context, destroy pthread locks. Safe to call with NULL.

uint32_t srhn4_add_node(SRHNNetwork4 *net, const char *label, NodeType type, LangID lang)

Create a new node, compute its embedding via srhn4_embed_text(), insert into the HNSW index, append a WAL entry. Thread-safe (acquires node write lock).

labelHuman-readable text. The embedding is computed from this string.
typeNODE_CONCEPT, NODE_PHYSICS, NODE_CODE, etc. Affects visualisation and LLM template selection.
langLANG_ENGLISH, LANG_HINDI, etc. Used for cross-language resonance discounting.
Returns: node ID (0-indexed), or UINT32_MAX on failure.
void srhn4_connect(SRHNNetwork4 *net, uint32_t a, uint32_t b, float weight, EdgeType type)

Create a bidirectional edge between nodes a and b. If the edge already exists, updates its weight. Grows neighbor arrays dynamically. Appends WAL entry. Weight must be in [0.001, 1.0] — clamped automatically.

SRHNResult4 srhn4_query(SRHNNetwork4 *net, const char *input)

The main entry point. Runs the complete 19-step pipeline. Thread-safe for concurrent callers — the LLM call is serialised via query_lock, graph reads use rwlock.

Returns: SRHNResult4 on the stack — copy it if you need to keep it after the next call.
void srhn4_feedback(SRHNNetwork4 *net, uint32_t query_id, float reward)

Submit a reward signal after a query. reward ∈ [-1.0, 1.0]. Triggers Hebbian weight update, contrastive signature update, entropy adjustment, and calibration table update.

query_idUse net->query_counter immediately after srhn4_query().
reward+1.0 = perfect, 0.0 = neutral, -1.0 = wrong/harmful response.

Embedding API

bool srhn4_embed_load_glove(SRHNNetwork4 *net, const char *path)

Load GloVe text-format vectors (e.g., glove.6B.300d.txt). Parses up to 400K word vectors, normalises to unit length, builds a hash table for O(1) lookup. Projects 300→384 via random orthogonal matrix. Call once before the first query.

Returns: true if at least 1 vector loaded. Network falls back to n-gram if not called.
float srhn4_resonance(const Signature *a, const Signature *b)

Cosine similarity between two unit-normalised 384-dim vectors, with cross-language discount (0.82×) and hash pre-filter. Returns [0, 1].

HNSW Index API

The HNSW index is built into SRHNNetwork4.hnsw and managed automatically. You only need these functions if building a custom index or doing bulk operations.

int srhn4_hnsw_search(HNSWIndex *idx, const Signature *query, uint32_t ef, uint32_t *result_ids, float *result_dists, int k, SRHNNetwork4 *net)

Search for k approximate nearest neighbours. ef controls recall/speed tradeoff — higher ef = better recall, slower search. Default: HNSW_EF_SEARCH = 64.

result_idsOutput array of node IDs, sorted by ascending distance.
result_distsOutput distances (1 - cosine_sim). Similarity = 1 - dist.
Returns: number of results found (≤ k).

Reasoning API

void srhn4_causal_path(SRHNNetwork4 *net, uint32_t from, uint32_t to, ReasoningChain *out)

Run Dijkstra's algorithm preferring EDGE_CAUSES/EDGE_CAUSAL edges (50% cost) and avoiding EDGE_CONTRADICTS. Stores the path in out with labels, relations, and per-hop strengths.

void srhn4_find_contradictions(SRHNNetwork4 *net, SRHNResult4 *result)

Scan all pairs of activated nodes with activation > 0.25 for EDGE_CONTRADICTS connections. Populates result.contradictions[]. Call after srhn4_propagate().

Memory API

void srhn4_working_mem_insert(SRHNNetwork4 *net, uint32_t node_id, float salience)

Insert a node into working memory (16 slots). If already present, updates salience. Evicts lowest-salience slot when full. Working memory nodes receive 10× activation bonus in propagation.

void srhn4_episodic_recall(SRHNNetwork4 *net, const Signature *query_sig, int k, Episode **out, int *n_out)

Find the k most semantically similar past episodes using mean pooling of their key node signatures. Applies a recency bonus that decays over hours. Used internally by srhn4_ctx_blend_query().

LLM Bridge API

bool srhn4_llm_init(LLMContext *llm, const char *model_path, int n_ctx, int threads)

Attempt to dlopen(SRHN_LLAMA_SO) and load the model. Returns true even if LLM unavailable — the system falls back to structured template responses. Set model_path = NULL to explicitly use fallback mode.

void srhn4_build_context(SRHNNetwork4 *net, SRHNResult4 *result, const char *query, char *out, int sz)

Build the structured context string that gets injected into the LLM prompt. Includes: top concepts with types, physics formulas, reasoning chains, contradictions, and recent conversation turns. You can call this independently to inspect what context the LLM receives.

Persistence API

bool srhn4_save(SRHNNetwork4 *net, const char *path)

Write a binary snapshot. Format: header (magic 0x53524833, version, counts) + node blobs (fixed + dynamic neighbor arrays) + hyperedges + lexicon + physics laws + stats + calibration table. After a successful save, the WAL file is truncated.

void srhn4_export_json(SRHNNetwork4 *net, const char *path)

Export graph as JSON for the visualizer. Includes: all non-pruned nodes with full metadata, all neighbour-pair edges and hyperedges, and stats block. The visualizer (srhn_v4_visualizer.html) reads this format directly.

Quick Reference — Critical Paths to Tune

Accuracy vs Speed: HNSW_EF_SEARCH
Graph size: PRUNE_THRESH + RESONANCE_THRESH
Multi-turn quality: WORKING_MEM_SLOTS
Reasoning depth: MAX_HOPS + SPIKE_DECAY
Memory footprint: EMBED_DIM + EPISODIC_CAP
Learning responsiveness: PAGERANK_INTERVAL

Adding a Domain

Walk-through: adding a Chemistry domain from scratch.

Step 1 — Add node loader in srhn_utils.c

c — srhn_utils.cvoid srhn4_load_chemistry(SRHNNetwork4 *net) {
    /* Helper: add a law-style node */
    static void (*add_law)(...) = ...;  // same pattern as srhn4_load_physics()

    /* Fundamental chemistry nodes */
    uint32_t atom   = srhn4_add_node(net, "atom element periodic table", NODE_CONCEPT, LANG_ENGLISH);
    uint32_t bond   = srhn4_add_node(net, "chemical bond covalent ionic", NODE_CONCEPT, LANG_ENGLISH);
    uint32_t react  = srhn4_add_node(net, "chemical reaction equilibrium", NODE_CONCEPT, LANG_ENGLISH);
    uint32_t ph_node= srhn4_add_node(net, "pH acid base hydrogen ion",   NODE_PHYSICS,  LANG_ENGLISH);

    /* Set physics-specific fields */
    if (ph_node != UINT32_MAX) {
        strncpy(net->nodes[ph_node].formula, "pH = -log₁₀[H⁺]", 127);
        strncpy(net->nodes[ph_node].units,   "dimensionless",     31);
    }

    /* Edges */
    srhn4_connect(net, atom,  bond,   0.85f, EDGE_CAUSAL);
    srhn4_connect(net, bond,  react,  0.80f, EDGE_CAUSES);
    srhn4_connect(net, react, ph_node,0.70f, EDGE_ASSOC);

    /* Causal hyperedge */
    uint32_t nodes[] = {atom, bond, react};
    srhn4_add_edge(net, nodes, 3, 0.8f, EDGE_CAUSAL,
        "atoms bond → molecular structure → chemical reactions");
}

Option C — Plug in your own embedding function

The embedding pipeline is designed for replacement. Override by modifying the srhn4_embed_text() function in srhn_embed.c. The required contract is:

c — contract// Input:  text string + language hint
// Output: unit-normalised float[EMBED_DIM] stored in out->vec[]
// out->magnitude must be 1.0f after normalisation
// out->hash = fnv1a(text) for fast pre-filter
// out->lang = lang parameter

void your_embed_function(const char *text, LangID lang, Signature *out) {
    /* Call your embedding model */
    /* Write results into out->vec[0..EMBED_DIM-1] */
    /* L2-normalise */
    float mag = 0.f;
    for (int i = 0; i < EMBED_DIM; i++) mag += out->vec[i]*out->vec[i];
    mag = sqrtf(mag) + 1e-8f;
    for (int i = 0; i < EMBED_DIM; i++) out->vec[i] /= mag;
    out->magnitude = 1.f;
}
Note — EMBED_DIM

If you change EMBED_DIM from 384, you must recompile everything. The HNSW index, resonance function, and all sig structs use EMBED_DIM at compile time. The projection matrix in srhn_embed.c will also need regenerating.

Connecting OpenAI-compatible APIs

You can replace srhn4_llm_generate() with an HTTP call to any OpenAI-compatible endpoint. The structured context built by srhn4_build_context() works as-is as the system message:

c — OpenAI-style replacement sketchbool srhn4_llm_generate_openai(LLMContext *llm, const char *prompt,
                               char *out, int max_out) {
    // Build JSON: {"model":"gpt-4o-mini","messages":[...]}
    // POST to http://localhost:11434/v1/chat/completions (Ollama)
    // or https://api.openai.com/v1/chat/completions
    // Parse response["choices"][0]["message"]["content"] into out
    return out[0] != '\0';
}

Fallback mode (no LLM)

When no LLM is loaded, srhn4_answer() calls the internal mock_generate() function which builds a structured response directly from the activated nodes, formulas, and reasoning chains. This is fully functional — useful for testing, embedded devices, or when privacy requires zero outbound calls.

Real-World Patterns

Pattern 1 — Offline Medical Decision Support

Deploy on a hospital server with no internet. Ingest drug interaction tables and clinical guidelines via POST /ingest. The CONTRADICTS edge type between treatment pathways provides the safety layer before any LLM response is delivered to clinical staff. The WAL ensures the knowledge base survives power cycles without re-ingestion.

Pattern 2 — Adaptive Tutoring System

Each student gets their own SRHNNetwork4 instance loaded from a binary snapshot. Call srhn4_feedback(net, query_id, +1.0) when the student answers correctly and -1.0 when wrong. The Hebbian learning gradually reshapes the activation weights toward topics the student understands, and away from topics they confuse. The episodic memory tracks which questions they've seen before.

Pattern 3 — IoT Knowledge Edge Node

Compile with make release (400KB binary). Flash to an ARM Cortex-A device. Pre-load domain knowledge at factory time via srhn4_save(). The device answers sensor queries locally with sub-10ms latency. The WAL write on every feedback call means field learning survives hard reboots.

Pattern 4 — RAG Pre-Filter for Cloud LLM

Use SRHN as the retrieval layer before calling GPT-4 or Gemini. Call srhn4_build_context() directly — this gives you the structured context string without triggering the built-in LLM. Inject that context into your cloud LLM prompt. You get SRHN's reasoning chains and contradiction detection for free, with cloud-quality text generation.

c — RAG pre-filter patternSRHNResult4 r = srhn4_query(net, user_query);

// Get the structured context SRHN built
char context[CONTEXT_BUF_SIZE];
srhn4_build_context(net, &r, user_query, context, sizeof(context));

// Send to your cloud LLM
your_openai_call(system_prompt=context, user_message=user_query, out=response);

Glossary

TermDefinition
SignatureA unit-normalised 384-dim float vector representing the semantic content of a node or query. Stored in Signature.vec[].
ResonanceCosine similarity between two Signatures. Range [0,1]. Values above RESONANCE_THRESH (0.12) are considered semantically related.
ActivationPer-node float [0,1] representing how "active" a node is in the current query context. Decays by 80% before each new query.
Spike decayThe factor (0.72) by which activation weakens at each BFS hop. Controls how far activation propagates from seed nodes.
Fast ringA priority set of up to 2048 nodes that receive a 1.15× resonance bonus and are never auto-pruned. Promoted via PageRank or manual calls to srhn4_promote_fast().
HNSWHierarchical Navigable Small World — a probabilistic graph structure enabling O(log n) approximate nearest-neighbour search over high-dimensional vectors.
Hebbian learning"Neurons that fire together wire together." When two nodes are co-activated and receive positive feedback, the edge between them strengthens by lr×reward×act_a×act_b.
PageRankIterative algorithm assigning importance scores to nodes based on the number and quality of connections. Run every 100 queries. Stored in node.importance.
Causal pathA directed path through the graph preferring EDGE_CAUSES/EDGE_CAUSAL edges, found via Dijkstra's algorithm. Represents an explainable chain of cause and effect.
ContradictionDetected when two activated nodes are connected by an EDGE_CONTRADICTS edge and both have activation > 0.25. Injected into the LLM prompt as a warning.
GroundingPost-generation verification that the LLM's response is consistent with the graph. Adds warning annotations if contradiction nodes appear in the response.
Working memory16-slot high-salience buffer. Nodes here receive 10× activation bonus in the next query. Decays 20% per query. Simulates short-term memory focus.
Episodic memoryCircular buffer of up to 10K past queries with key node IDs, confidence, and reward. Recalled by cosine similarity with recency bonus.
WALWrite-Ahead Log. Appends CRC32-protected operations (add_node, add_edge, update_weight, prune) before they're applied. Replayed on crash recovery.
Self-growWhen a query has low resonance (<0.18), SRHN creates new nodes for novel tokens in the query, connecting them to existing near-neighbours via HNSW.
CalibrationIsotonic regression table that maps raw resonance scores to real accuracy probabilities, fitted from feedback history. Enforces monotonicity via PAV algorithm.
Context blendingThe effective query signature is a weighted blend of the current query plus the last 4 turns (weights: 0.25, 0.125, 0.0625, 0.03125). Enables multi-turn coherence.
Entropy scorePer-node float [0,1] combining usage rate, recency, and PageRank importance. Nodes below PRUNE_THRESH (0.08) with low usage are candidates for pruning.
